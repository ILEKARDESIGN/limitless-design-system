Grouped / Clustered Bar Chart
Best for → Side-by-side category comparison · Budget categories per period
Bars placed side by side within each group, not stacked. Essential for comparing absolute values of Core vs CB vs Capital per quarter without cumulation. Each bar in the cluster uses a distinct neon colour with 2px gap between bars and wider gap between groups.

100% Stacked Bar Chart
Best for → Proportion comparison · Share shifts over time · Composition changes
All bars are the same height (100%), segments show proportional share. Answers "what percentage was Core vs CB vs Capital?" regardless of total amounts. Inline percentage labels in each segment. Only the top segment gets border-radius.

Bubble Chart
Best for → Three-variable comparison · Cost vs outcome vs volume
Scatter plot where circle radius encodes a third variable (participant count, hours, budget). X = cost, Y = outcome quality, size = volume. Neon strokes with 20-25% opacity fills for layering. Bebas Neue labels inside each bubble. Quadrant dividers at 4% opacity for "high value / low cost" analysis.
Service Value AnalysisProvider ROIPortfolio Strategy
Stepped Line Chart
Best for → Discrete state transitions · Status changes · Rate schedules
Flat horizontal segments that jump vertically at each change point. Perfect for plan status transitions (Intake → Draft → Review → Active), NDIS hourly rate changes, or any data that holds a value until it suddenly changes. Dots at each transition point mark the exact change moment.

Streamgraph
Best for → Organic flow visualisation · Volume evolution · Editorial narratives
Flowing organic stacked area chart centred on a baseline, creating a symmetric river-like shape. Each stream's width encodes volume. More visually distinctive than a regular stacked area — ideal for narrative/editorial use. Pink, cyan, yellow, purple layers at descending opacity.
Service VolumeEditorial ReportsNarrative Dashboards
Horizon Graph / Multi-Row Sparkline Dashboard
Best for → Dense multi-series monitoring · Caseload overview · Compact time-series
Compressed layered bands showing multiple time-series in minimal vertical space. Each row is a participant with a dual-opacity area fill: lighter band for overall range, darker band for recent intensity. Four participants in the space a single line chart would use. Also covers circle packing as an alternative hierarchical composition view.

Step Progress Indicator
Best for → Onboarding flows · Intake processes · Multi-stage workflows
Numbered horizontal circles connected by lines showing where a workflow currently sits. Completed steps: pink filled. Current step: yellow with glow. Future steps: outline only at diminishing opacity. Connector lines are solid pink (completed) or dim (pending). Every intake, onboarding, and approval flow needs this.

User Journey / Process Flow Diagram
Best for → End-to-end process mapping · Decision points · Path visualisation
Horizontal node-arrow chain showing a participant's path through a process with duration annotations and decision branches. Completed nodes: solid neon fills. Current: yellow. Future: dashed outline. Orange branch for waitlist/exception paths. Different from a timeline (date-based) and Sankey (volume-based) — this shows the logical process structure with optional branching.

Empty State
Best for → Zero-data widgets · First-use experience · Onboarding prompts
What a widget shows when there's no data to display. The most overlooked pattern — every chart, table, board, and list needs one. Uses the infinity symbol as a subtle watermark, clear descriptive text explaining what would appear here, and a primary action button to create the first item. The empty state IS the onboarding.

LOADING — SHIMMER SKELETON
Loading Skeleton / Shimmer Placeholder
Best for → Data fetch in progress · Perceived performance · Layout stability
Animated shimmer placeholders that mirror the shape of the incoming content — circles for avatars, bars for text, rectangles for cards. The shimmer gradient sweeps from left to right on a 1.5s loop. Shows the user exactly what's coming and prevents layout shift. Uses 3–6% opacity white with a sliding 6% highlight band.

Error State
Best for → Failed data fetches · API errors · Connection issues
What a widget shows when data loading fails. Pink exclamation circle, descriptive error message, technical error code in JetBrains Mono, and a retry action button. Distinct from empty state (no data exists) — error state means data exists but couldn't be retrieved. Every component needs both.
