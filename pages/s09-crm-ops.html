Caseload Table with Check-In Recency
Best for → Full caseload overview · Engagement monitoring · Overdue detection
The master caseload view. Each row shows participant name (bold white), NDIS number (monospace), assigned coordinator, complexity tag (pink/orange/standard), last check-in with freshness indicator, and plan status badge. The check-in column is the critical differentiator from a generic table: green circle = recent (under 7 days), orange triangle = due (8–30 days), pink triangle = overdue (30+ days). Overdue rows get a subtle pink background tint. Sortable by any column, filterable by coordinator, complexity, and check-in status.
Check-In Freshness System
● Cyan dot = contacted within 7 days (healthy) · ▲ Orange triangle = 8–30 days (attention needed) · ▲ Pink triangle = 30+ days (overdue, row tinted) · Elapsed time in matching accent colour · Bold weight on overdue values · Row background tint at 3% for flagged items

Tag / Badge Management Strip
Best for → Disability type encoding · Complexity flags · Quick-scan participant attributes
Coloured tag pills displayed across the top of a participant profile, encoding disability type, complexity tier, and support needs at a glance. Each tag uses a semantically meaningful colour (red = ABI/medical, blue = communication, pink = high needs, cyan = mobility, dark = complexity). The "+" button opens a tag selector for adding new tags. Tags are removable on click. This is a data presentation pattern — it replaces paragraphs of disability description with scannable visual indicators.

Implementation Progress Table
Best for → Line-item budget tracking · Claiming progress · Financial accountability
Per-line-item implementation tracking showing what's been claimed against what was funded. Each row has: description, total amount, implemented (claimed) amount in accent colour, remaining in dimmed text, and an inline progress bar. Total row with bold weight and top border. This is the most critical financial view in any NDIS CRM — it shows exactly where money has gone at the support-item level. Progress bars use category-specific colours: pink = ADL, cyan = SCP, yellow = fully utilised. The "Add Implementation" button at top-right enables booking new service agreements against budget lines.
